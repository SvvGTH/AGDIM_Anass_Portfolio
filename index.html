Projet CAP 4A – Lampadaire urbain autonome piézoélectrique

Depuis janvier 2025 NOVA / ESIEA ESP32 • ESP-NOW Node.js • Express • MySQL Agile • Jira

Conception d’un système embarqué de récupération d’énergie (20 capteurs piézoélectriques) pour alimenter un lampadaire LED, avec détection de présence (PIR) et batterie Li-ion.

Contexte & objectifs

  • Récupérer et gérer l’énergie produite pour maximiser l’autonomie.
  • Assurer une détection fiable de présence et un pilotage LED adapté.
  • Fournir un monitoring Web (tableau de bord, alertes, rôles).

Processus

  • Organisation en équipe selon une approche agile : backlog, planification, sprints hebdomadaires, suivi via Jira.
  • Revues régulières, répartition claire des responsabilités : électronique, firmware, back-end, front-end.
  • Intégration progressive : mesures → calculs énergie → communication → supervision.

Réalisation technique

  • Firmware ESP32 (C/Arduino) : mesure tension/courant via ADC 12 bits, calcul énergie produite/consommée, pilotage lampadaire.
  • Communication ESP-NOW vers une passerelle ESP32-B.
  • Plateforme Web : Node.js, Express, MySQL, HTML/CSS/JavaScript (dashboard, alertes, gestion de rôles).

Résultats (à valoriser)

  • Chaîne complète : capteurs → microcontrôleur → communication → supervision.
  • Traçabilité projet via backlog/sprints et revues régulières.
  • Base solide pour itérations (optimisation énergie, fiabilité, UX monitoring).
